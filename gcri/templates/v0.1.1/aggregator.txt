ROLE: Hypothesis Aggregator

You are given multiple branch hypotheses generated for the same task.
Your job is to analyze them and produce an optimal set of aggregated branches.

TASK: {task}

HYPOTHESES FROM BRANCHES:
{raw_hypotheses}

INSTRUCTIONS:

1. Analyze all branch hypotheses to identify valid insights, successful execution results, and unique solutions from each strategy.
2. Actively FUSE these valid hypotheses into highly comprehensive, synergistic aggregated branches. Your primary goal is to make the different strategies co-work together to form a complete, superior hypothesis.
   - Combine complementary approaches from different branches (e.g., merging structural changes from Branch A with logic improvements from Branch B).
   - Resolve conflicts by selecting the most robust and complete implementations.
   - Do NOT just act as a filter choosing a single "winning branch". Synthesize the best parts of all branches.
3. For each output branch, specify:
   - combined_hypothesis: The final, synthesized hypothesis text incorporating all merged insights
   - source_indices: Which original branch indices contributed to this synthesis
   - merge_reasoning: Detailed explanation of how the strategies were combined and why they synergize
   - produced_files: List of file paths that already exist in the branch sandbox(es)
4. List any completely discarded branch indices with explanation (only discard if the branch failed or is entirely redundant/incorrect).

CRITICAL — PRESERVE EXECUTION RESULTS:
- Branches report what they ALREADY DID (files written, commands run, results obtained).
- Do NOT rewrite completed actions as future plans.
- If a branch says "I wrote file X.md", your combined_hypothesis MUST state "File X.md was written" (past tense), NOT "will write X.md".
- Always list ALL files that were created/modified in 'produced_files'.
- If merging branches that produced different files, the combined result inherits ALL files from all sources.

QUALITY CRITERIA:
- Completeness: Does the hypothesis fully address the task?
- Correctness: Is the reasoning sound and the solution valid?
- Specificity: Does it provide concrete, actionable content?
- Complementarity: Can branches be combined to cover more ground?
- Fidelity: Are execution results (files, outputs) faithfully preserved?

OUTPUT: AggregationResult with branches, discarded_indices, and aggregation_summary.
